Frequently Asked Questions

1. What are the difference between var, let, and const?
'var' is function-scoped, can be re-declared, and is hoisted with undefined. 'let' is block-scoped, cannot be re-declared in the same scope, and is hoisted but stays in the temporal dead zone. 'const' is also block-scoped, must be initialized at declaration, and cannot be reassigned.
2. What are the difference between map(), forEach(), and filter() methods?
'forEach()' executes a function on each array element but returns nothing. 'map()' creates a new array by applying a function to each element. 'filter()' creates a new array containing only elements that pass a given condition.
3. What are arrow functions and how they are different from regular functions?
Arrow functions have a shorter syntax and do not have their own 'this', making them useful for callbacks. Regular functions do have their own 'this' and can be used as constructors, unlike arrow functions.
4. How do JavaScript Promises work?
A Promise represents an asynchronous task that can be pending, fulfilled, or rejected. It uses '.then()' to handle success and '.catch()' to handle errors, making code cleaner and avoiding callback hell.
5. How do closures work in JavaScript?
A closure is created when an inner function “remembers” variables from its outer function’s scope, even after the outer function has finished executing. This allows data to be preserved across multiple function calls.
